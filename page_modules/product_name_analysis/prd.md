# PRD: 제품명 분석

## 목적

제품명과 원재료명을 입력받아 식품 표시 규정 관점에서 제품명을 분석한다.
전처리 후 6종의 세분화 분석을 수행하고, 결과를 패턴 단위로 취합하여 결론 텍스트를 생성한다.

## 입력 index 목록

| key | label | multiline | required | 비고 |
|---|---|---|---|---|
| product_name | 제품명 | false | true | 한 줄 텍스트 |
| ingredients | 원재료명 | true  | true | 쉼표/괄호 포함한 긴 텍스트 |

## LLM 프로세스 단계

1. **전처리** — 제품명/원재료명 정규화 (공백/특수문자/괄호 처리 등).
2. **세분화 분석 (6종)** — LLM 으로 6가지 관점에서 병렬/순차 분석.
   - 분석1, 분석2, 분석3, 분석4, 분석5, 분석6 (실제 항목은 추후 확정)
3. **(최종 분석 생략)**
4. **취합/결론** — 6종 결과를 패턴 단위로 Python 로직에서 취합하여 최종 텍스트 생성.

## 결과 구조

- `intermediates`:
  - `preprocess`: 전처리된 제품명/원재료명
  - `seg_1` ~ `seg_6`: 세분화 분석 6종 각각의 결과
- `final_text`: 취합된 결론 텍스트

## 특이사항

- 세분화 6종의 실제 분석 항목은 추후 정의. 현재는 placeholder (분석1 ~ 분석6) 로 둠.
- 취합은 LLM 이 아닌 **Python 로직(패턴 기반)** 으로 구현.
- 최종 분석 단계는 사용하지 않음 (세분화 → 바로 취합).

## 전처리 Prompt
# 목표
식약처의 제품명 판단기준 적합성 분석을 위해, 입력된 제품명에 대한 정보를 세분화하고자 합니다.
아래 지시사항을 이용하여 입력된 <data>에 대해 분석하세요.

# 입력데이터
제품명: {제품명}
원재료: {원재료명}

# 지시사항
목표: 제품명 분해 및 분류
**제품명만** 의미 단위(명사, 형용사 등)로 분해하고, 각 단위를 아래 중 하나로 분류합니다.

※ 중요: 분해 및 분류 대상은 오직 "제품명"뿐입니다. "원재료명"은 분해하거나 분류하지 마십시오. 원재료명은 제품명의 각 토큰이 실제 원재료에 해당하는지 확인하기 위한 참조 데이터로만 사용합니다.

  (A) 원재료명 또는 원재료명의 일부 → 함량 표시 대상
  (B) 성분명 → 함량 표시 대상
  (C) 통칭명 (과일, 해산물, 견과 등) → 함량 표시 대상
  (D) 식품유형명 / 즉석섭취ㆍ편의식품류명 / 요리명 → 함량 표시 면제
  (E) "향" 또는 "맛"이 붙은 표현 (예: "딸기향", "우동맛") → 별도 규칙 적용
  (F) 수식어ㆍ형용사 등 판단 불필요 (예: "얼큰", "맛있는", "진한", "특제", "매운")
  (G) 상호ㆍ브랜드명 

주의사항:
- "~맛"이 붙은 경우(예: "튀김우동맛"): "~맛"은 향미를 나타내는 표현이므로 원재료명이 아닌 (E)로 분류합니다. "우동맛"에서 "우동"을 원재료로 판단하지 마십시오.
- 요리명과 원재료명이 동시에 해석 가능한 경우(예: "조개탕"): 전체 단어가 요리명으로 통용되면 요리명(D)으로 판단합니다.
- "곰탕", "설렁탕", "칼국수", "짜장면", "우동", "카레", "파스타", "라멘" 등은 요리명/식품유형명(D)에 해당합니다.
- "~차", "~음료" 등 접미사 앞의 단어가 원재료명일 수 있으므로 반드시 분리하세요.
- **"숫자+통칭명" 패턴 (예: "16곡", "10곡", "5종야채", "7가지과일")**: 숫자는 수식어(F)로, 뒤따르는 통칭명은 (C)로 분류합니다. "곡"은 곡물/곡류를 통칭하는 표현이므로 반드시 (C) 통칭명으로 분류하십시오.
- 복합어의 경우 분해에 주의하십시오:
  · "파곰탕쌀국수" → "파" + "곰탕" + "쌀" + "국수". "곰탕"은 요리명(D), "국수"는 식품유형명(D), "파"는 원재료명(A), "쌀"은 원재료명(A).
  · "콩국수" → "콩" + "국수". "콩"은 원재료명(A),"국수"는 식품유형명(D).
  · "밀싹보리차" → "밀싹" + "보리" + "차". "밀싹"은 원재료명(A), "보리"는 원재료명(A), "차"는 식품유형명(D). 
  · "대추차"→"대추"+"차" . "대추"는 원재료명(A), "차"는 식품유형명(D). 
  · "튀김우동맛" → "우동"은 식품유형명/요리명(D), "맛"은 향미 수식어(E).
  · "특제라멘" → "특제"는 수식어(F), "라멘"은 요리명(D).
  · "야채수"→"야채"+"수" . "야채"는 통칭명(C), "차"는 식품유형명(D). 
  · "16곡미숫가루" → "16"은 수식어(F), "곡"은 통칭명(C), "미숫가루"는 식품유형명(D).

# 지시사항
## 1단계: 분석 대상 여부 판단
- 제품명에 원재료명/성분명이 **포함되지 않은 경우**
  . result = "해당없음" (분석대상 아님)
  . keyword_list, err_keyword_list, passed_list 필드는 빈 값으로 둠
- 제품명에 원재료명/성분명이 **포함된 경우**
  . result = "해당"
  . 2단계로 진행

## 2단계: 원재료별 규칙 준수 여부 판단 (result가 "해당"인 경우에만)
- [규칙] 제품명에 사용된 원재료명/성분명은 원재료 목록에 표시되어야 한다.
- 제품명에 포함된 각 원재료명/성분명에 대해 개별적으로 판단:
  . 원재료 목록에 존재하는 경우 → `passed_list` 필드에 해당 원재료명 기재
  . 원재료 목록에 존재하지 않는 경우 → `err_keyword_list` 필드에 해당 원재료명 기재
- 제품명에 여러 원재료가 포함된 경우, 각각을 독립적으로 판단하여 해당 필드에 나열

** 주의사항
  . 주표시면 데이터는 판단에 고려하지 않음
  . 함량 및 비중은 고려하지 않음
  . 성분명의 경우, **가루와 같은 표현도 허용함 (예: "메밀가루"는 "메밀"과 동일하게 취급)
  . 부분 매칭: 제품명의 용어가 원재료 목록의 더 긴 항목에 포함되면 대응 인정 (예: "자몽" → "레드자몽농축액")

반드시 아래 JSON 형식으로만 응답하세요:
```json
{
  "result": "해당없음 또는 해당",
  "reason": "(자유 기술)",
  "keyword_list": "",
  "err_keyword_list": "",
  "passed_list": ""
}
```

각 필드 설명:
- result: 법률 해당 여부 (해당없음: 분석대상에 포함되지 않는 경우, 해당: 분석대상에 포함되는 경우)
- reason: 판단 근거 요약 (자유롭게 서술)
- keyword_list: 제품명에서 추출된 전체 원재료/성분/요리명 리스트 (제품명에 그대로 기술된 표현)
- err_keyword_list: 입력데이터의 원재료 목록에 없는 제품명에서 추출된 원재료/성분/요리명 리스트 (제품명에 그대로 기술된 표현)
- passed_list: 준수한 원재료 리스트 (원재료 목록에 기술된 원재료명으로 기술)

## 세분화 분석-2 프롬프트
# 목표
식약처의 제품명 판단기준 적합성 분석을 위해, 입력된 제품명에 대한 정보를 세분화하고자 합니다.
아래 지시사항을 이용하여 제품 데이터에 대해 분석하세요.

# 입력데이터
제품명: {제품명}
원재료: {원재료명}

# 제품 상세 정보
{PRE_out}

# 지시사항

## 1단계: 분석 대상 여부 판단
- 제품 상세 정보(PRE_out)에서 (C) 통칭명으로 분류된 토큰이 **하나도 없는 경우**
  . result = "해당없음" (분석대상 아님)
  . keyword_list, err_keyword_list, passed_list 필드는 빈 값으로 둠
- 제품 상세 정보(PRE_out)에서 (C) 통칭명으로 분류된 토큰이 **하나라도 있는 경우**
  . result = "해당"
  . 2단계로 진행

※ 중요: PRE_out에서 이미 (C)로 분류된 결과를 신뢰하십시오. "통칭명의 범위가 좁다", "단일 원재료를 가리킨다", "숫자가 붙어 있어 특정된다" 등의 이유로 해당 여부를 재해석하지 마십시오.
- "고기"는 돼지고기·소고기·닭고기 등 여러 식육 원재료를 통칭하는 명칭입니다.
- "곡"은 곡물/곡류의 줄임말로, 쌀·보리·밀·옥수수 등 여러 곡물 원재료를 통칭하는 명칭입니다. "16곡", "10곡", "7곡" 등 숫자가 앞에 붙어도 "곡"은 여전히 통칭명입니다.
- "과일", "해물", "야채", "견과" 등도 동일하게 통칭명에 해당합니다.

※ 절대 규칙: PRE_out에서 (E)로 분류된 토큰이 존재하면, 어떤 이유로든 "해당없음"을 반환해서는 안 됩니다. 이 판단을 번복하지 마십시오.
- "살구향", "딸기향", "포도향", "멸치맛", "바나나맛" 등 "특정 원재료명+향" 또는 "특정 원재료명+맛" 형태는 모두 (E)에 해당하며, "일반적 향 표현"이라는 분류는 존재하지 않습니다.
- "매운맛", "순한맛", "달콤한맛" 등 특정 원재료명이 포함되지 않은 일반적 미각 표현만이 해당없음 대상입니다. 이들은 PRE_out에서 (E)가 아닌 (F)로 분류됩니다.

## 2단계: 맛/향 관련 규칙 준수 여부 판단 (result가 "해당"인 경우에만)
- [규칙] "맛" 또는 "향"을 내기 위하여 사용한 원재료로 향료만을 사용하여 제품명 또는 제품명의 일부로 사용하고자 하는 때에는 원재료명 또는 성분명 다음에 "향" 자를 사용하고, 제품명 주위에 "합성OO향 첨가(함유)" 또는 "합성향료 첨가(함유)" 또는 "향료" 등의 표시를 하여야 한다. 다만, 해당 원재료의 "맛" 또는 "향"을 내기 위해 향료물질로 사용한 것이 합성향료물질로만 구성된 것에 한한다.
- 제품명에 포함된 각 맛/향 관련 명칭에 대해 개별적으로 판단:
  . (E)로 분류된 토큰에서 원재료명을 추출 (예: "살구향" → "살구", "딸기맛" → "딸기")
  . 원재료 목록에서 해당 원재료의 실제 원물(살구, 딸기 등)이 사용되었는지 확인
  . 해당 원재료의 맛·향을 내기 위해 향료만 사용되었는지 확인
  . 향료만 사용된 경우: 제품명에 "향" 자가 붙어있고 "합성OO향 첨가" 등의 표시가 있으면 → `passed_list` 필드에 기재
  . 위 조건을 충족하지 못하면 → `err_keyword_list` 필드에 기재

** 주의사항
  . 주표시면 데이터는 판단에 고려하지 않음
  . 함량 및 비중은 고려하지 않음

```json
{
  "result": "해당없음 또는 해당",
  "reason": "(자유 기술)",
  "keyword_list": "",

```

각 필드 설명:

- keyword_list: 제품명에서 추출된 전체 키워드 리스트 (제품명에 그대로 기술된 표현)

- passed_list: 준수 원재료 리스트 (원재료 목록에 기술된 원재료명으로 기술)
