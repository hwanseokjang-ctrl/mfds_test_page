## 목적

LLM 기반 분석 결과를 테스트할 수 있는 데모페이지를 구현하고자 합니다.
금번에 만들 페이지는 식약처의 식품표시AI 과업에 대한 테스트를 목적으로 하고 있습니다.

분석항목별로 분석 탭이 늘어날 예정이며, 각 분석항목마다, 입력 및 분석 프로세스와 결과 구조가 서로 다릅니다.

기본적으로는 입력은 여러 index 단위로 텍스트 입력을 받습니다. 예를 들어, 제품명, 원재료명, 업종, 소비기한, 영양정보, 주의사항 등이 있습니다.

LLM 프로세스는 크게 전처리 > 세분화 분석 > 최종 분석 > 취합 및 결론 TEXT 생성으로 이루어집니다. 분석에 따라 세분화 분석이나 최종 분석이 생략될 수도 있습니다. 최종 결론은 Text로 생성되며, 중간 프로세스 결과도 확인할 수 있도록 제공하려고 합니다.

## 개발 방법

기본적인 페이지 틀은 공통으로 만들며, 각 분석항목이 추가될 때마다 페이지를 메뉴로 추가하려고 합니다. 

각 페이지별 개발 내용은 기본 틀은 유지하되 별도로 개발하며, 공통으로 활용할 함수는 같이 활용할 수 있도록 처리합니다. 

페이지별로 별도 요구사항 정리할 수 있도록, 초기 페이지 개발과 개별 페이지 개발을 구분하여 개발 가능하도록 폴더를 구성합니다.

그리고 각 폴더에 요구사항을 prd.md 형태로 정리한뒤(파일명은 바뀔 수 있음), 이를 기반으로 각 페이지를 개발하는 방식으로 진행하려고 합니다. 

* 일단 첫번째 페이지는 제품명분석을 목적으로 만들려고 합니다.
* 제품명 분석은 제품명과 원재료명을 입력으로 받고, 프로세스는 전처리 > 세분화 분석(6종) > 취합하여 텍스트 생성(패턴 단위. python 구현) 으로 진행됩니다.
